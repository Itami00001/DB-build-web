Оформление readme проекта git
1. Структура readme
1. Название и описание проекта (Project Title and Description)
Начните с краткого названия, которое точно отражает суть вашего проекта. За ним должно
следовать подробное описание проекта. Что делает ваш проект? Какие проблемы он
решает? Почему он ценен?
2. Инструкции по установке (Installation Instructions)
В вашем файле README должны быть четкие инструкции по установке. Если у проекта
есть какие-либо зависимости, обязательно укажите их в шагах. Фрагмент кода,
демонстрирующий команды установки, может быть очень полезен.
npm install your-package
3. Использование (Usage)
Покажите примеры использования вашего проекта. Пользователь должен понимать, как
реализовать ваш проект с минимальными усилиями. Включите фрагменты кода или
скриншоты для иллюстрации.
const yourPackage = require('your-package');
yourPackage.doSomething();
4. Вклад (Contributing)
Если ваш проект является проектом с открытым исходным кодом и вы хотите побудить
других к участию, включите раздел о том, как можно внести свой вклад. Укажите им на
необходимые ресурсы и процессы.
5. Документация (Documentation)
Если ваш проект имеет дополнительную документацию, предоставьте ссылки на эти
ресурсы.
6. Лицензия (License)
Включите раздел, посвященный лицензии вашего проекта. Это информирует
пользователей об их законных правах и ограничениях при использовании вашего
программного обеспечения.
7. Контактная информация (Contact Information)
Укажите контактные данные или способы поддержки проекта в случае возникновения
вопросов или проблем.
8. Благодарности (Acknowledgments)
Если ваш проект использует код из других проектов с открытым исходным кодом,
профессионально будет указать источник и отдать должное.
2. Синтаксис
Полное описание приведено на странице https://docs.github.com/ru/get-started/writingon-github/getting-started-with-writing-and-formatting-on-github/basic-writing-and-formattingsyntax
2.1 Заголовки
Для создания заголовка добавьте от одного до шести символов # перед текстом
заголовка. Количество используемых символов # определит уровень иерархии и размер
шрифта заголовка.
# A first-level heading
## A second-level heading
### A third-level heading

2.3 Цитирование кода
Вы можете ввести код или команду в предложение, используя одинарные обратные
кавычки. Текст внутри обратных кавычек форматироваться не будет. Вы также можете
нажать сочетание клавиш Command+E (Mac) или Ctrl+E (Windows/Linux), чтобы вставить
обратные кавычки для блока кода в строку Markdown.
Use `git status` to list all new or modified files that haven't yet been
committed.
Результат
Для форматирования кода или текста в отдельный блок используйте тройные обратные
кавычки.
Some basic Git commands are:
```

git add
git commit
```
