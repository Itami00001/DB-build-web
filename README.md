# Build-Shop - Строительная фирма с системой объявлений

## Описание проекта

Build-Shop - это веб-приложение для строительной фирмы, которое позволяет пользователям покупать и продавать строительные материалы с использованием внутренней криптовалюты C-coin. Проект реализован с использованием Node.js, Express, Sequelize, PostgreSQL и Docker.

## Основные функции

### 🏗️ Сущности проекта
- **Пользователи** - регистрация, авторизация, роли (user/admin)
- **Категории материалов** - иерархическая структура категорий стройматериалов
- **Материалы** - каталог строительных материалов с характеристиками
- **Объявления** - пользователи могут создавать объявления о продаже материалов
- **Заказы** - система заказов с разными статусами
- **Транзакции** - перевод C-coin валюты между пользователями
- **Корзины** - корзина покупок с возможностью оформления заказа
- **Отзывы** - система отзывов и рейтингов

### 💰 C-coin валюта
- Внутренняя валюта для транзакций
- Администратор может выдавать C-coin пользователям
- Создание объявления стоит 10 C-coin
- Пользователи могут обмениваться валютой

### 👤 Пользовательские роли
- **User** - обычный пользователь с полным доступом к функционалу
- **Admin** - администратор с доступом к панели управления БД

### 🛠️ Админ панель
- Просмотр статистики базы данных
- Просмотр всех таблиц БД в удобном формате
- Экспорт данных в PDF
- Просмотр логов приложения
- Доступ к Swagger документации

## Установка и запуск

### Требования
- Docker и Docker Compose
- Node.js 18+ (для локальной разработки)

### 1. Клонирование репозитория
```bash
git clone <repository-url>
cd build-project
```

### 2. Настройка переменных окружения
Скопируйте файл `.env` и при необходимости измените параметры:
```bash
cp .env.example .env
```

Основные параметры в `.env`:
- `POSTGRESDB_USER` - пользователь PostgreSQL
- `POSTGRESDB_ROOT_PASSWORD` - пароль PostgreSQL
- `POSTGRESDB_DATABASE` - имя базы данных
- `POSTGRESDB_LOCAL_PORT` - порт PostgreSQL на хосте
- `NODE_LOCAL_PORT` - порт приложения на хосте

### 3. Запуск с помощью Docker Compose
```bash
docker-compose up -d --build
```

Эта команда:
- Создаст и запустит контейнер с PostgreSQL
- Создаст и запустит контейнер с pgAdmin 4
- Создаст и запустит контейнер с Node.js приложением
- Выполнит миграции базы данных
- Загрузит начальные данные

### 4. Проверка работы
- **Приложение**: http://localhost:3000
- **API документация**: http://localhost:3000/api-docs
- **pgAdmin**: http://localhost:5050

## Транзакции и уровни изоляции

Проект использует транзакции ACID для обеспечения целостности данных:
- **Atomicity** - атомарность операций
- **Consistency** - согласованность данных
- **Isolation** - изолированность транзакций
- **Durability** - долговечность данных

Уровень изоляции по умолчанию: `READ COMMITTED`

## Развертывание

### Production
1. Измените переменные окружения в `.env`
2. Используйте `docker-compose.prod.yml` для production конфигурации
3. Настройте reverse proxy (nginx)
4. Включите HTTPS
5. Настройте бэкапы базы данных

### Docker команды
```bash
# Пересборка с кешем
docker-compose build --no-cache

# Просмотр логов
docker-compose logs -f app

# Остановка
docker-compose down

# Полная очистка
docker-compose down -v
```

## Поддержка

При возникновении проблем:
1. Проверьте логи контейнеров: `docker-compose logs`
2. Убедитесь, что все порты свободны
3. Проверьте переменные окружения в `.env`
4. Очистите Docker кеш при необходимости
