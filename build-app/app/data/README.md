# Начальные данные для проекта Build-Shop

Эта папка содержит SQL файлы и скрипты для автоматической загрузки начальных данных при развертывании проекта на новом устройстве.

## Файлы

### SQL файлы
- `01_users.sql` - Начальные данные пользователей (test, test1, test2, admin)
- `02_categories.sql` - Начальные данные категорий материалов (6 категорий)

### Скрипты
- `init-data.js` - Скрипт для автоматической загрузки начальных данных через seeders

## Как это работает

При запуске `docker-compose up --build`:

1. Dockerfile копирует папку `app/data` в контейнер
2. docker-compose.yml выполняет команду: `node app/data/init-data.js && npm run dev`
3. Скрипт `init-data.js` проверяет наличие данных в базе:
   - Если база пустая - загружает начальные данные через seeders
   - Если данные уже есть - пропускает загрузку

## Развертывание на новом устройстве

Для развертывания проекта на новом устройстве:

1. Клонируйте репозиторий
2. Создайте файл `.env` с настройками базы данных
3. Запустите: `docker-compose up --build`
4. Данные загрузятся автоматически при первом запуске

## Ручная загрузка данных

Если нужно перезагрузить данные вручную:

```bash
docker exec build-project-app-1 node app/data/init-data.js
```

## Примечания

- Скрипт использует существующие seeders из папки `app/seeders`
- Данные загружаются только если база пустая
- Для полной перезагрузки данных нужно очистить базу данных
